# QuestForge AI: UI Concept & Element Explanation

본 프로젝트는 사용자의 목표(Goal)를 구체적인 실행 계획(Roadmap)으로 변환해주는 **"어드벤처 게임"** 컨셉을 가지고 있습니다. 단순한 도구를 넘어, 사용자가 자신의 목표를 정복해야 할 '퀘스트'로 느끼게 하는 것이 핵심입니다.

---

## 1. 핵심 컨셉 (Core Concept)
- **Quest (퀘스트)**: 사용자의 목표를 단순히 '할 일'이 아닌, 보상과 성장이 있는 모험으로 정의합니다.
- **The Oracle (예지자)**: AI와 대화하는 과정을 예지자로부터 계시를 받는 과정으로 비유하여, 목표를 구체화하는 지루한 과정을 흥미로운 인터랙션으로 바꿉니다.
- **Blueprint (설계도)**: 대화를 통해 추출된 핵심 정보들을 '성전의 설계도'처럼 시각화하여 정보의 결여를 직관적으로 알게 합니다.
- **Readiness Level (준비도)**: 퀘스트를 시작하기 위한 최소한의 정보가 수집되었는지를 게임의 '경험치(XP)'와 '레벨' 개념으로 표현합니다.

---

## 2. 주요 UI 요소 설명

### [Discovery View] (목표 탐색 단계)
1. **Chat Panel (Oracle Session)**
   - 예지자와의 대화창입니다. 사용자가 목표를 말하면 AI가 질문을 던지며 목표를 구체화합니다.
   - **의미**: 퀘스트의 배경과 조건을 설정하는 '스토리텔링' 단계입니다.
2. **Score Gauge (Readiness Level)**
   - 대화 내용의 구체성을 점수화(0-100 XP)하여 보여줍니다.
   - **의미**: 정보가 충분히 모였는지(레벨 1~4)를 시각적으로 보여주며, 사용자가 더 상세히 답변하도록 유도합니다.
3. **Blueprint Preview (Quest Journal)**
   - 대화에서 추출된 5대 핵심 요소(목표, 이유, 기간, 장애물, 자원)를 보여줍니다.
   - **의미**: 모호한 대화 속에서 '팩트'만 추출하여 정리된 '저널' 역할을 합니다.

### [Visualization View] (계획 시각화 단계)
1. **Quest Map (Roadmap)**
   - 사용자의 목표를 Milestone(이정표) - Step(단계) - Task(과업)의 계층 구조로 시각화합니다.
   - **의미**: 정복해야 할 경로를 한눈에 보여주는 '전략 지도'입니다.
2. **Template Switcher (View Modes)**
   - Quest Map, Mountain, Mandalart, Gantt 등 네 가지 뷰를 제공합니다.
   - **의미**: 사용자의 선호에 따라 계획을 다각도에서 조명합니다.
3. **Quest Intel (Node Detail Panel)**
   - 특정 단계를 클릭하면 나타납니다. 구체적인 Action Item과 AI에게 전략(Tactical Advice)을 물어볼 수 있는 전용 챗이 있습니다.
   - **의미**: 해당 퀘스트를 클리어하기 위한 구체적인 '전술 가이드'입니다.

---

## 3. 개선 제안 (Proposed Improvements)

### 🧩 Mandalart 뷰의 고도화 (현재 가장 개선이 필요한 부분)
- **현재**: 단순히 3x3 그리드에 노드를 나열하고 있습니다.
- **개선**: 
  - 중앙에 '핵심 목표'를 두고 주변 8개 칸에 'Milestone'들을 배치하는 **진정한 만다라트 레이아웃**을 적용해야 합니다.
  - 각 Milestone을 클릭하면 다시 그 Milestone을 중심으로 8개의 상세 Task가 펼쳐지는 '줌인/아웃' 효과를 추가하면 더욱 강력한 시각화가 가능합니다.

### 🎮 게임 요소(Gamification) 강화
- **XP 애니메이션**: Readiness Level이 올라갈 때 레벨업 사운드나 파티클 효과를 추가하여 성취감을 극대화합니다.
- **난이도 설정**: 퀘스트의 성격에 따라 'Easy/Normal/Hard/Boss' 등급을 부여하여 시각적인 분위기를 바꿀 수 있습니다.

### 🔍 인터랙티브 가이드
- **Oracle의 제안**: Blueprint Preview에서 비어있는 항목(예: 자원 부족)이 있을 때 오라클이 먼저 "Loot(자원) 정보가 부족합니다. 어떤 장비가 필요한가요?"라고 넛지를 주는 기능을 강화합니다.
